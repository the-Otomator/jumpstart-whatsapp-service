🛒 Provisioning Flow — New Customer Purchase

1
Customer lands on purchase/signup page
Self-service or via referral. Chooses WhatsApp Service plan.
2
Payment via SUMIT
Customer enters credit card. SUMIT processes payment and fires webhook.
→ SUMIT webhook → automation endpoint
3
Org provisioning (automated)
Webhook triggers creation of:
• Supabase: new row in central_subscriptions with generated org_id
• JumpStart: new business owner account with WhatsApp module unlocked
→ Supabase INSERT + JumpStart account creation
4
Customer receives login + setup instructions
Email with JumpStart Hub credentials. Links to connect their WhatsApp device.
→ Email via SUMIT or automation
5
Customer connects WhatsApp in Hub
Opens WhatsApp module → clicks "Connect device" → scans QR in iframe from wa.otomator.pro/connect/:orgId
→ JumpStart Hub → WhatsApp Service
6
Instance appears in Otomator Admin
Nizan sees the new session in admin panel. Can monitor health, disconnect, manage subscription.
→ Otomator Admin reads WhatsApp API + Supabase
